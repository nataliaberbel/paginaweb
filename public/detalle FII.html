Introducción al desarrollo web:

Este tema nos presenta el desarrollo web, comenzando por la diferencia entre un sitio web y una página web, la cual se encuentra en que la primera es un documento que se muestra en un navegador web, mientras que la segunda se refiere a una ubicación central que contiene múltiples páginas web. Tras ello, entramos en contacto con los sitios webs, en los cuales el usuario sólo puede desplazarse, y una aplicación web, la cual permite una mayor interacción, como realizar pagos.

Al operar la arquitectura cliente-servidor se ejecutan diversas operaciones. Comienzan cuando el cliente solicita una página web a través de un dispositivo habilitado para la red. Tras esto, el servidor web recibe y procesa la solicitud, y finalmente envía una respuesta al cliente. En la construcción de una aplicación web vamos a encontrarnos con el frontend y el backend. El primero es lo que los usuarios ven, y tan solo contituye un 20% de todo el código. Este abaraca los lenguajes de programación empleados por el navegador. Por otro lado, el otro 80% está constituido por el backend, la parte que el usuario no sabe que está.

El lenguaje estándar para programación de los documentos mostrados en el navegador web es HTML, el cual también suele ser acompañador por CSS y lenguajes como JavaScript. HyperText se refiere a un "texto dentro del texto", y es una forma de vincular dos o más páginas web. Por otro lado, Markup Language es un lenguaje empleado para aplicar las convenciones de diseño y formato a los documentos de texto. Cascading Style Sheets es un lenguaje de hojas de estilo utilizado para describir la presentación de un documento escrito en HTML o XML. Proporciona una vista mucho más cuidada, aplicando los estilos como reglas en casacada a cada aparición de los elementos.

La estructura básica de una página HTML se compone de diferentes bloques que definen la estructura de la página web. Las etiquetas HTML son como palabras clave que definen cómo el navegador web formateará y mostrará el contenido. Un sitio web o una aplicación web se compone de diferentes archivos que generalmente se distribuyen en diferentes carpetas. Esto significa que es importante definir las rutas para encontrar los diferentes archivos. Una ruta absoluta describe la ubicación desde el directorio raíz del sistema de archivos. Una ruta relativa describe la ubicación de un archivo en relación con el directorio.
